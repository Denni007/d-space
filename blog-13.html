Understanding GA4 BigQuery Schema in 2025

Google Analytics 4 (GA4) provides a powerful integration with BigQuery, allowing businesses to analyze their website and app data at a granular level. Understanding the GA4 BigQuery export schema is essential for making data-driven decisions.

Overview of GA4 BigQuery Export Schema

GA4 exports raw event data to BigQuery, where each event is recorded as a row in a table. The schema is structured to support event-driven analytics and provides detailed insights into user interactions.

Key Components of GA4 BigQuery Schema

1. Event Tables
  • events_YYYYMMDD: Daily tables where each row represents an event (e.g., page_view, purchase) that occurred on your site or app.
  • events_intraday_YYYYMMDD: Real-time event data for the current day, allowing up-to-the-minute analysis.
2. User Tables
  • pseudonymous_users_YYYYMMDD: Stores data for users identified by pseudonymous identifiers, such as device IDs.
  • users_YYYYMMDD: Contains information on users with assigned user IDs, used when authentication is implemented.
3. Nested Fields

GA4 uses a nested and repeated schema, which means some fields contain arrays. Key nested fields include:

  • event_params: An array of key-value pairs providing additional context to events, such as page titles or button labels.
  • user_properties: Custom attributes assigned to users, such as language preferences or membership status.

Conclusion

GA4’s BigQuery integration provides businesses with detailed and flexible analytics capabilities. By understanding the GA4 BigQuery schema, businesses can effectively analyze user behavior, optimize marketing campaigns, and improve decision-making. Leveraging BigQuery’s powerful querying capabilities ensures efficient data handling and actionable insights.
