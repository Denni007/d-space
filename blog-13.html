Understanding GA4 BigQuery Schema in 2025

Google Analytics 4 (GA4) works seamlessly with BigQuery, allowing businesses to analyze website and app data in depth. Understanding the GA4 BigQuery export schema helps businesses track user interactions and improve data-driven decision-making.

How BigQuery Imports Your GA4 Data

When you link GA4 to BigQuery, your analytics data is stored in your BigQuery project, which holds datasets that contain raw analytics data. The dataset follows this structure:

analytics_ → Contains multiple tables categorized into event and user data.

Projects, Datasets, and Tables

  • Projects: The top-level container where GA4 data is stored.
  • Datasets: A collection of tables related to your GA4 property.
  • Tables: Store event, user, and session data.

Category 1: Event Tables

Daily Event Export

Your GA4 event data is stored in tables named events_YYYYMMDD, where YYYYMMDD represents the date of the recorded data.

Example: If you linked GA4 to BigQuery on March 1, 2025, your first table will be events_20250301.

Key Features:

  • Each row represents a user interaction (event) on your site or app.
  • Only events recorded after linking GA4 to BigQuery are available.
(Almost) Real-Time Event Export

The events_intraday_YYYYMMDD table stores real-time event data for the current day.

Note: This table is only available if Streaming Export was enabled when linking GA4 to BigQuery.

How Event Data is Structured in BigQuery

GA4 tracks every interaction as an event, which contains multiple details:

  • event_name: The type of event (e.g., page_view, purchase).
  • event_timestamp: The time the event occurred.
  • event_params: Extra data related to the event (e.g., page title, button clicked).
  • traffic_source: Identifies the source of the traffic (e.g., Google, Facebook).

Each event is stored as a record with nested fields for better data organization.

Category 2: User Tables

Unlike event tables, user tables store data about individual users rather than their interactions.

Types of User Tables:

  • users_YYYYMMDD: Stores data for logged-in users with User IDs.
  • pseudonymous_users_YYYYMMDD: Stores data of users identified by Device IDs (anonymous users).

These tables allow businesses to analyze returning users, customer journeys, and behavior trends more effectively.
